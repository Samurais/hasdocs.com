HasDocs is a polyglot cloud documentation platform. You can sync the repositories from GitHub and build documentation using generators such as Sphinx, Jekyll, and Javadoc.

Step 1: Sign up

Sign up using your existing GitHub account. HasDocs requires permission to read from your public and private repos.

Step 2: Write the docs

Write the documentation for your project. HasDocs currently supports Sphinx, Jekyll, and Javadoc as documentation generators.

You can also bypass the generator altogether to write the html yourself.

Step 3: Import a repository

Sync your GitHub repositories by clicking the sync button on your account page. Then install a a post-receive hook by clicking the create hook button.

Choose the programming language the repo is in and the generator to use for building the documentation.

Then specify the location of documentation relative to the project root. For Python projects, virtualenv will install the dependencies if an optional requirements file is specified.

Step 4: Deploy the docs

Deploy the docs by pushing to GitHub. Your docs are built by the selected documentation generator and then uploaded to HasDocs server.

Documentations are updated every time you push to GitHub.

For private repositories, docs are visible only to people with appropriate permissions.
